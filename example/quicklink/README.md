# quicklink (sample repo)

A tiny self-hosted URL shortener — **not a real product**, but a working instance of the
Spec-Driven Development kit used to dogfood it. The interesting part is [`specs/`](specs/),
not the code.

Start here: read [`CLAUDE.md`](CLAUDE.md), then [`specs/product.md`](specs/product.md). The
specs are fully filled in (including a real rejected-alternatives log in
[`specs/decisions.md`](specs/decisions.md)) so you can see what "good" looks like before
filling the blank templates in `../../kit/`.

The code (`app/`, `tests/`) is just enough to make [`specs/repo-map.md`](specs/repo-map.md)
point at real files.
